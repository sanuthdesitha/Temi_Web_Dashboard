  ☐ linux/run.sh created and executable (chmod +x)

  ☐ static/ directory present with CSS, JS, images
  ☐ webview templates included (9 HTML files)
  ☐ Documentation files present:
    - QUICK_START.txt

    - README_LINUX.txt

Python Files Included:
  ☐ app.py (main Flask application)

NETWORK AND CONNECTIVITY VERIFICATION
================================================================================

Local Access:
  ☐ http://localhost:5000 works
  ☐ http://127.0.0.1:5000 works
  ☐ Browser shows no SSL/certificate warnings

Remote Access (if deploying on network):
  ☐ Find server IP address (ipconfig on Windows, ip addr on Linux)
  ☐ Access from another computer: http://<server-ip>:5000
  ☐ Works from multiple browsers
  ☐ Works from multiple devices on network
  ☐ Firewall configured to allow port 5000

MQTT Connectivity Verification
================================================================================

Local MQTT Broker:
  ☐ If using local broker, verify it's running
  ☐ Test connectivity: mosquitto_sub -h localhost -t # (or equivalent)
  ☐ Configure in Settings: MQTT Host = localhost, Port = 1883

Remote/Cloud MQTT Broker:
  ☐ Broker address accessible from deployment machine
  ☐ Port accessible (usually 1883 or 8883 for TLS)
  ☐ Firewall allows outbound connection to MQTT port
  ☐ Configure in Settings with broker details

MQTT Broker Test:
  ☐ Go to Settings page
  ☐ Enter MQTT broker details
  ☐ Test connection button returns success
  ☐ No connection timeout errors

================================================================================

  ☐ Go to Settings → Robot Management
  ☐ Add test robot with:
    - Name: "TestRobot" (or actual robot name)
    - Serial: Actual robot MQTT serial number
    - Home Base: "home" or actual home waypoint name
  ☐ Click Save
  ☐ Robot appears in robot list
  ☐ Can select robot in dashboard

Robot Status:
  ☐ Select robot in dropdown
  ☐ Request battery status (if robot is running and connected)
  ☐ Status updates in real-time
  ☐ No timeout errors

Waypoints Configuration:
  ☐ Add at least 3 test waypoints in Map Management
  ☐ Each waypoint has unique name and coordinates
  ☐ Can save and retrieve waypoints
  ☐ Database persists waypoints across restarts
